使用hexo个人总结

2020-11-18

hexo也使用了挺久的了，虽然只是偶尔使用。。（☆☆）

在这对使用过的命令做一个简单的总结，以便以后查看

  • 创建新文章 hexo new "文章名"(会自动创建.md文章和相同文件名的文件夹，用来存放图片等)

  • 添加图片使用![](输入图片地址)方式添加图片（使用Typora软件可直接使用自带编辑器，很方便，但是图片格式地址必须写成如下格式，反斜杠也不能写错）

  • 使用 <!--more-->来对首页文章只显示一部分内容（要注意，more之前的内容需要再复制到more下面，否则查看全文只显示more后面的内容）

  • 使用hexo generate 生成静态文件

  • 使用hexo deploy 部署网站

  • 使用hexo server 启动服务器。默认情况下，访问网址为： http://localhost:4000/

    （当使用hexo s 后，对文章修改，保存后直接同步到网站，但是仍然需要完整步骤才能更新到服务器）

  • 使用hexo clean 清除缓存文件 (db.json) 和已生成的静态文件 (public)。

    在某些情况（尤其是更换主题后），如果发现对站点的更改无论如何也不生效，可能需要运行该命令。

    总结：先使用hexo g 然后hexo d 最后hexo s
